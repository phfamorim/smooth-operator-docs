Install

Blender 5.1 or newer is required. The SDF grid nodes this tool is built on don't exist in older versions — Blender will refuse to enable it there, with a version message.

  1. Extension (recommended): drag smooth_operator-1.1.0-extension.zip into the Blender window and confirm — or go to Edit → Preferences → Get Extensions → Install from Disk and pick the zip.
  2. Legacy add-on: Edit → Preferences → Add-ons → Install, pick smooth_operator.zip, then tick Smooth Operator in the list.

Once enabled, the tools live in the 3D View sidebar (N key) under the Smooth Operator tab.

Quick start

  1. Select one or more mesh objects.
  2. Press Remesh in the Smooth Operator tab. Each object gets a live modifier with a Voxel Size chosen automatically from its dimensions — safe on any mesh.
  3. Dial Voxel Size down for more detail, and Smoothness up or down to taste. Everything updates live; nothing is destructive.
RULE OF THUMB

Detail thinner than about two voxels disappears — that's how distance fields work. If a thin part vanishes, lower the Voxel Size before touching any filter.

The modifier, socket by socket

Main settings

PRE-SMOOTHING LOWPOLY INPUT

To round faceted lowpoly input before it hits the voxel grid, put a plain Subdivision Surface modifier above the remesh — Blender's own, creases and all. (Earlier builds had a Subdivide input inside the modifier; it was removed in favor of the native one, and old files get a real Subdivision Surface modifier inserted automatically on upgrade.)

SettingDefaultWhat it does
Voxel SizeautoThe resolution of the remesh, in scene units. Halving it roughly quadruples the face count and compute time. The Remesh button sets it to ~1/100 of the object's largest dimension.
Adaptivity0Merges flat areas into larger polygons at extraction time. 0 keeps the uniform, bake-safe grid. For quality low-poly, prefer the Decimate button instead (see below).
Combine ElementsonOn: the whole object is treated as one solid — overlapping shells weld together. Off: every loose part is remeshed on its own and overlapping parts stay separate.
Smooth NormalsonWrites custom normals sampled from the SDF gradient — shading with no faceting and no voxel ripple. Turn off on any remesh that gets boolean'd afterwards (the stacking button handles this for you), or the cut faces will shade black in Cycles.

Filters panel

OpenVDB level-set filters. They reshape the distance field itself, before the mesh exists — which is why they leave no tool marks on topology.

FilterDefaultThe job
Smoothness2The main polishing pass (gaussian blur of the field). 2 is the classic OpenVDB look; higher polishes more but erodes thin detail first.
Despike0Median filter — removes spikes, needles and scan junk while keeping large edges crisp. 1–2 cleans up messy booleans and 3D scans without softening the whole model. Each iteration also erodes curved regions slightly, so high values pull the surface inward — a look, not a bug.
Filter Width1Kernel radius (in voxels) for the two filters above — Smoothness and Despike. Width 2 smooths roughly like triple the iterations, in a single pass. Relax and Concave Smooth are not affected.
Relax0Mean-curvature flow — eases the tightest bends first (pinch lines, seams). Softens convex and concave alike; detail loss goes global above ~4.
Concave Smooth0Adds material only in concave corners, rounding boolean seams into fillets while convex edges stay untouched. Self-limiting — it converges instead of running away.
Inflate/Deflate0Fatten (positive) or erode (negative) the whole surface by a real-world distance. Stay within about two voxels of zero, or the surface leaves the field's narrow band and the mesh breaks up.

Safety panel — the crash guard

Every voxel remesher has the same horror story: a Voxel Size typo — 0.0001 on a 2-meter prop — and Blender freezes with your unsaved work. Smooth Operator clamps a Voxel Size finer than the surface can afford to a safety floor, inside the modifier itself. The mesh still generates (just capped) and an orange warning appears on the modifier. Fix the value, or raise the cap deliberately:

SettingDefaultWhat it does
Max Resolution2000The face budget behind the guard — a catastrophe net, not a taste police. The default (~24 million faces) is high enough that deliberate fine-voxel work passes untouched, so the same Voxel Size always means the same physical voxels on every object; only typo-level requests get clamped. When a clamp does engage, the warning states the effective size and the Max Resolution needed for your requested one. Lower the budget on weaker machines, raise it for extreme density.
SCOPE, HONESTLY

The guard covers Voxel Size — the classic remesher freeze. It won't rescue a 50-million-poly boolean elsewhere in your stack, or a Subdivision Surface modifier cranked on a dense mesh.

Workflow buttons

Remesh

Adds the modifier to every selected mesh, with a safe automatic Voxel Size. Pressing it on an object that already has a Smooth Operator does something smarter — see the boolean sandwich.

Visibility All / Visibility Selected — On | Off

Toggles every Smooth Operator modifier in the scene (or just on the selected objects) in the viewport only — renders are untouched. Park the heavy remeshes while you model; bring them back when you're done.

Preview Mode

A fast viewport that stays one click from bake quality. While the toggle is on, the last Smooth Operator of every stack gets the preview Adaptivity (default 0.1, slider under the toggle) — flat areas merge into big polygons, cutting the face count hard — and every earlier pass stays at 0. Only the last pass matters: adaptivity anywhere else is paid for and then remeshed away. Stacking a new remesh hands the preview to the new last pass automatically.

Turning it off restores Adaptivity 0 everywhere — the uniform, bake-safe grid. Collapse always bakes final quality, ignoring Preview Mode.

BEFORE A FINAL RENDER

Preview adaptivity is a real modifier value — it applies to renders too, including on machines without the add-on installed. Turn Preview Mode off before final renders and before shipping a .blend.

Decimate

Adds a pre-tuned Decimate (Collapse) modifier at the bottom of the stack. A uniform remesh plus quadric decimation gives cleaner low-poly than Adaptivity does — the error-driven collapse spends polygons where curvature needs them, and the smooth normals and materials ride along. Pressing it again retunes the existing Decimate instead of stacking another.

TIP

Decimate re-evaluates on every upstream tweak. Disable it in the viewport while dialing the remesh, or Collapse first and decimate the baked copy.

Collapse

Apply-everything, but reversible: bakes a copy of each selected object with all modifiers applied, files it into a collection (default "Collapsed Meshes" — the checkbox and name field next to the button control this), and hides the procedural original in place. Un-hide the original any time to keep editing. Materials and custom normals survive the bake.

The boolean sandwich

The workflow this tool was built around: remesh, cut, remesh again.

  1. Press Remesh — smooth the base shape.
  2. Add your Boolean modifier above it. Use the MANIFOLD solver — it preserves loose-part structure where EXACT would weld the operand's own shells.
  3. Press Remesh again. The button notices the existing pass and stacks a second one: settings copied from the first, and Smooth Normals moved to the final pass — mid-stack custom normals corrupt boolean cut faces, which is what renders black in Cycles.
  4. Raise Concave Smooth on the final pass to round the boolean seams into fillets.
DELETING A PASS

Remove the last Smooth Operator and the previous pass takes over its role automatically — Smooth Normals comes back (keeping the deleted pass's on/off choice) and the preview Adaptivity moves over. A pass that still has a boolean after it is never given normals (that would render black in Cycles), and deleting middle passes or renaming modifiers changes nothing.

Bake-down pipeline

UVs and vertex data do not survive remeshing — the surface is rebuilt from scratch; that's true of every voxel remesher. Material assignments transfer automatically. The intended finishing pipeline:

  1. Collapse the finished object to a baked copy.
  2. Decimate (before or after Collapse) to your target polycount — the gradient normals survive.
  3. Re-unwrap the baked copy, then bake normals from it (or from the full-res version onto it). The SDF-gradient shading bakes cleanly into normal maps — that's what it's for.

Shortcuts

Every panel button ships unbound. To assign keys: Edit → Preferences → Add-ons → Smooth Operator — each button has an editable shortcut field, bound into the 3D View keymap. (Blender's right-click "Assign Shortcut" doesn't work for custom-prefix operators — use the add-on preferences instead.)

Upgrading & file portability

  • Files stay portable. The remesher is a geometry-node group embedded in your .blend — scenes open, evaluate and render on machines that don't have the add-on installed. The panel buttons are the only thing that needs the install.
  • Old files self-upgrade on open. After updating the add-on, just open an old file: every existing Smooth Operator modifier migrates to the new version automatically, settings intact. Appended objects that bring their own copy of the node group are consolidated too. (Pressing Remesh also triggers the same upgrade, if the file was opened before the add-on was updated.)
  • Updates are safe to install over the previous version — no need to uninstall first.

Troubleshooting

Boolean'd areas render black in Cycles
A remesh below the boolean has Smooth Normals on — its baked custom normals corrupt the new cut faces. Turn Smooth Normals off on every pass except the last, or just use the stacking Remesh button, which does exactly that automatically.
An orange warning appeared on the modifier
The crash guard engaged: your Voxel Size exceeded what the mesh can afford and was clamped. The result on screen is the capped version. Raise Max Resolution in the Safety panel if the density was intentional.
Thin parts disappear
Anything thinner than ~2 voxels can't be represented. Lower the Voxel Size (the guard has your back if you overshoot), and prefer Despike over Smoothness when cleaning junk near thin geometry.
The mesh breaks apart when inflating/deflating
Inflate/Deflate beyond about two voxels leaves the field's narrow band. Keep it small, or increase Voxel Size alongside it.
Open or non-manifold input gives odd results
A distance field represents closed, watertight surfaces. Open shells get thickened or partially eaten depending on voxel size — cap holes first for reliable conversion.
It feels slow on a huge mesh
The remesher runs on Blender's geometry-nodes evaluator (CPU). Reference: a 129k-triangle prop updates in ~70 ms on a 13th-gen desktop Intel. If you're far off that, check for a Subdivision Surface modifier cranked on dense input, turn on Preview Mode while you model, park other remeshes with the Visibility buttons, and disable Decimate while dialing.
My Adaptivity value keeps changing on its own
Preview Mode manages the Adaptivity of the last pass (and zeroes the others) whenever it's toggled or its value changes. Turn Preview Mode off if you want to drive Adaptivity by hand — though for quality low-poly, the Decimate button beats Adaptivity anyway.
Renders look decimated / flat-shaded in patches
Preview Mode was left on — its adaptivity applies to renders too. Turn it off before final renders.
